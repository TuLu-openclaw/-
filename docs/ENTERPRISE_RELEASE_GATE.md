# 星枢影视企业版官方级上线门槛

目标：项目必须对标企业版/官方版质量，禁止“小作坊低仿”。任何正式发布必须同时满足产品、前端、后端、移动端、TV端、CI发布和可观测性要求。

## 1. 总原则

- 官方级体验：页面结构、交互、动效、错误提示、加载状态必须像正式商业产品，不允许拼凑感。
- 真实数据链路：所有分类、频道、筛选、分页、详情、播放都必须来自真实接口或真实页面解析。
- 禁止假兜底：不能用其他分类、假数据、硬编码列表冒充未接入功能。
- 后端兜底优先：移动端/TV端不得依赖浏览器 CORS 或公网临时代理作为主链路。
- 可诊断：失败必须显示可读原因，并在日志/构建产物中能追踪。
- 可回归：每次发布必须有明确构建日志、产物下载路径、版本号和验证记录。

## 2. 前端官方级标准

### 2.1 信息架构

- 首页必须按原站模块分区展示，不得把所有内容压成一个普通网格。
- 频道、子频道、专题、榜单、时间线、搜索、筛选、分页必须保留原站结构逻辑。
- 未接入功能必须隐藏或标注“未接入真实接口”，不得做假入口。

### 2.2 视觉体验

- 统一设计系统：颜色、字号、间距、圆角、阴影、按钮、卡片、弹窗必须一致。
- 桌面、手机、TV 分别有独立布局策略，禁止桌面布局硬塞进手机/TV。
- 关键页面必须有加载态、空态、错误态、重试入口。
- 暗色模式、高对比度、低动效模式不能破坏主要操作。

### 2.3 手机端

- 不允许横向溢出。
- 所有按钮/输入/卡片可触控区域不低于 40px，核心操作建议 44px+。
- 频道/筛选/标签过多时必须横向滑动或折叠，不得挤压变形。
- 播放器必须适配竖屏、横屏、安全区和系统返回。
- 失败提示必须能看懂：哪个接口、哪个源、为什么失败、是否可重试。

### 2.4 TV端

- 必须支持 Leanback 入口。
- 必须支持方向键焦点、确认键、返回键。
- 焦点高亮必须远距离可见。
- 网格字号、卡片间距、播放器控制必须适合遥控器操作。
- 不允许必须依赖鼠标悬停才能完成核心操作。

## 3. 后端官方级标准

- Android/TV 可调用的核心命令不得被 `#[cfg(target_os = "windows")]` 排除。
- 影视请求必须优先走 Tauri/Rust 后端，避免 WebView CORS 和公网代理不稳定。
- 每个资源站 adapter 必须有：列表、搜索、详情、播放、分页、错误处理。
- 加密/签名/解密逻辑必须跨平台可编译，不能依赖 Windows-only 外部命令。
- 外站请求必须有合理超时、User-Agent、Referer/Origin、错误分类。
- 播放地址解析必须区分直播、点播、广告片段、失效链接。

## 4. 播放器上线标准

- 支持 HLS、MP4、iframe 兜底。
- HLS 错误必须自动重试有限次数，并给出可读失败原因。
- 支持线路切换、选集、续播、下一集。
- 手机端播放器全屏适配安全区。
- TV端播放器可用遥控器控制。
- 禁止播放广告/预览 VOD 冒充实时直播。

## 5. CI / 发布标准

正式发布前必须通过：

- `npm run build`
- `cargo fmt --check`
- Android CI：APK + AAB 构建成功。
- TV manifest patch 日志存在：Leanback + touchscreen optional。
- Release 页面能下载 APK/AAB。
- 构建日志保存到本地或 artifact，用户不需要在网页里盲找。

建议增加：

- Android 构建前执行 Rust Android `cargo check` 或等价 Tauri build 校验。
- 手机/TV 布局 Playwright 或截图测试。
- 关键接口 smoke test：列表、搜索、详情、播放地址解析。

## 6. 上线禁止项

出现以下任一项，禁止标记正式上线：

- Android/TV 构建失败或没有可下载产物。
- 手机端核心页面横向溢出、按钮不可点、播放器不可用。
- TV端无遥控器焦点或 Leanback 入口缺失。
- 后端核心命令仅 Windows 可用，移动端调用失败。
- 用假分类/假兜底冒充原站结构。
- 构建日志、失败原因、下载路径无法追踪。
- 只在桌面浏览器测试，未验证 Android/TV。

## 7. 当前已知风险

- 本机缺 Android Rust target，不能直接本地验证 Android Rust 编译。
- 本机缺 clang/MSVC 编译链，不能直接本地完整 `cargo check`。
- Playwright 浏览器内核未安装，暂不能自动截图验证手机布局。
- 当前本地修复未提交推送，远端 CI 不包含本轮补丁。

## 8. 正式上线流程

1. 完成本地修复。
2. 跑 `npm run build`、`cargo fmt --check`。
3. 提交并推送。
4. 触发 Android Phone/TV CI。
5. 下载 APK/AAB 和完整构建日志。
6. 手机端安装测试：启动、导航、搜索、列表、详情、播放、错误重试。
7. TV端安装测试：入口、遥控器焦点、导航、播放、返回。
8. 记录验收结果后，才允许标记正式上线。
